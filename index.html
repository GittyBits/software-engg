Software vs Program

S. no. Software Program
1) A software is a collection of programs or instructions to execute a task. A program is a set of instructions to perform a specific task.
2) It consists many lines of code. It can be a single line of code.
3) It is written by many users. It can be written by single user.
4) Software can be a single program. A single program cannot be a software.
5) It can be further classified into system & application software. It is not classified further.
6) It is costly to develop. It takes low cost to develop.
7) Software development requires well-planning, organising & time management. There is no need of planning to develop program.
8) Software contains code & documentation. Program contains code.
9) Examples: Windows, MS Office, Android. Examples: Web browser, malware.

Q. What are the different software models? Also explain SQA Point, Deliverables, and Milestones.

CMM (Capability Maturity Model), Waterfall Model, Prototyping Model etc. are different types of software models created to standardize software.

  • SQA Point:
    is Software Quality Assurance policy. We check bugs in software part by part.

  • Deliverable:
    The time range/estimate in which software is delivered to the user/client.

  • Milestone:
    The time range/estimate in which software is delivered to the company.

Q. Explain the Capability Maturity Model (CMM) with its different levels.

  • In CMM software is made using these 5 levels.
  • CMM was proposed by SEI (Software Engineering Institute).
  • Initial (L1) → Repeatable (L2) → Defined (L3) → Managed (L4) → Optimised (L5)

Levels of CMM

  1. Level 1: Initial Stage

    • Planning what software to make, Preparing feature list, Deciding cost and budget,Setting tentative timeline, Creating DFD Level 0, Making flowcharts and use case diagrams

    • Few processes are defined and success depends on individual effort.

  2. Level 2: Repeatable Stage

    • Focuses on Software Requirement Specification (SRS).
    • Basic project management processes are established.
    • Process are established to track core schedule functionality
    • The necessary process discipline is placed to repeat earlier successes on similar projects/applications.

  3. Level 3: Defined Stage

    • The software process is defined for both management and engineering activities.
    • Documented, standardized, and integrated into a standard software process.
    • Approved version of the process is used for developing and maintaining software.

  4. Level 4: Managed Stage

    • Detailed measures of software process and product quality are collected.
    • Organization sets quantitative quality goals for both software projects and maintenance.

  5. Level 5: Optimised Stage

    • Focus on continuous process improvement.
    • Uses quantitative feedback and advanced techniques.
    • Key feature: specifying and continually improving performance through feedback, incremental changes, and innovative practices.

Q. Explain the Waterfall Model. Write its advantages and disadvantages

  • Can be used in small-scale projects, but fails in larger ones.
  • Drawback: Each step must be finished before moving to the next.
    Example: If we are in the design stage, we cannot accept a new requirement.

Phases :-

  1. Requirements
    • Requirements given by company/client.
  2. Analysis
    • Involves cost calculation, hardware requirements, etc.
  3. Design
    • Planning the making of the product.
    • Includes SRS (Software Requirement Specification).
  4. Coding & Implementation
    • Dividing modules into chunks.
    • Different development teams work on separate parts and later integrate them.
  5. Testing
    • Includes:
      • Unit Testing (small things like errors, pointers, etc.)
      • Integration Testing
      • System Testing
      • Acceptance Testing
  6. Operational Deployment
    • Occurs after code is ready.
    • Includes release notes, terms & conditions, documentation.
  7. Maintenance
    • Fixing problems, improving system, upgrades, etc.

Q. Explain the Prototype Model. Mention its standard steps and applications.

  • Used when requirements are not well specified and are likely to change.
  • Involves building a prototype of the system and refining it through customer feedback.

Working flow :

Prototype Module ↔ User
(Customer feedback → Develop/Refine Prototype → Testing of prototypes by customers → Results/Info → Refined prototype)

Standard Steps

  1. Requirement Gathering & Analysis
  2. Quick Design
  3. Build a Prototype
  4. Initial User Evaluation
  5. Refining Prototype
  6. Implement Product & Maintain

Q. Explain the Iterative Waterfall Model. How is it different from the basic Waterfall Model?

  • Feedback can be taken at any stage (e.g., after requirement analysis, design, or coding).
  • Allows moving back to earlier phases to incorporate changes.
  • Reduces drawbacks of the basic Waterfall Model where no changes were allowed once a stage was completed.

Difference from Waterfall Model

  • Waterfall Model: Linear, no backtracking once a stage is complete.
  • Iterative Waterfall Model: Feedback loops exist, making it more flexible and adaptive.

Explain the Incremental Model in SDLC with a neat diagram. How does it differ from the Prototype Model?

  • The system is developed in increments (parts/modules) and delivered step by step.
  • Example:
    • Part A → Student attendance
    • Part B → Teacher attendance
    • Part C → Student records
      (expanding the same product)

  1. Requirement analysis occurs only once.
    • After that, the product is built incrementally (part by part).
    • Unlike the Prototype Model where requirement analysis occurs repeatedly after feedback.
  2. Less workload in each stage.
    • Each increment has smaller design and testing tasks.
    • More time-efficient than building the whole system at once.

Q. Explain evolutionary model.

  • It is a combination of Iterative and Incremental Models
  • Feedbacks occurs at every stage with every build.

Q. Explain the Spiral Model. What are its main phases? Also state its advantages and disadvantages.

  • Used in most companies and considered one of the most efficient models.
  • Involves risk assessment at every stage.
  • Model can be visualized as a spiral with 4 quadrants.

Phases of Spiral Model

  1. Customer Communication
    • Involves requirement analysis.
    • Interaction with customers to gather needs.
  2. Planning
    • Done by the team to determine resources, timelines, cost, etc.
  3. Risk Analysis
    • Involves foreseeing possible risks (e.g., module failures, data loss).
    • Risk analysis reports help in identifying drawbacks and preparing mitigation strategies.
  4. Engineering
    • Coding, development, and testing of modules.
  5. Construction & Release
    • Creating working product, installation, deployment, and support.
  6. Customer Evaluation
    • Continuous feedback from customers after each release.

Advantages

  • Handles risk effectively.
  • Continuous customer feedback.
  • Suitable for large and complex projects.
  • Flexible for changes.

Disadvantages

  • High cost.
  • Risk analysis requires expert knowledge.
  • More complex compared to linear models.

Q. Explain the V-Model. What does it mean by verification and validation? Where is this model used?

  • V stands for Verification & Validation.
  • Each development stage has a corresponding testing phase.
  • Testing and validation occur at every stage.
    • Example: Requirement analysis undergoes acceptance testing before moving to system design.

  • Ensures verification and validation at every step.

  • Used in critical systems like:

    • RTOS (Real-Time Operating Systems)
    • CT scans, health analysis systems
    • Space research
      (where errors can have severe consequences)

  • With each stage, testing moves to smaller scale:

    • Module < Architecture < System
    • Example: Flat/Room < Floor < Building

Advantages

  • Errors detected early due to continuous testing.
  • Very reliable for safety-critical systems.
  • Provides structured development and validation.

Disadvantages

  • Costly and time-consuming.
  • Rigid, not suitable for projects with changing requirements.

Q. Explain the RAD (Rapid Application Development) Model. What are its main features?

  • Used when there is a time constraint for developing software.
  • Involves parallel development by multiple teams.
  • Uses a combination of models like:
    • Iterative
    • Incremental
    • Prototype

Advantages

  • Time-efficient; faster product delivery.
  • Encourages user involvement and feedback.
  • Parallel module development reduces workload.

Disadvantages

  • Requires large skilled teams.
  • Higher cost.
  • Not suitable for small projects.
